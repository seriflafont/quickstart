Defaults

  • includes bootstrap (please remove what you don't need!)
  • includes jquery
  • uses handlebars for templates
  • js is written in es6 and transpiled using rollup
  • compiles templates and serves the pages with express.js
  • includes starter template and component files
  • includes starter js and sass files
  • structured as an AEM project for easier handoff

Instructions

  • Make sure you rename the following: the project name in package.json, and the 2 directories named 'quickstart'
  • The frontend folder contains the necessary files for serving the pages via express.js.
  • The gruntfile is set up to manage local development, but includes tasks for the maven build on AEM.
  • The pom.xml file includes a maven task that will call the grunt frontend build tasks necessary for AEM. This includes pulling down node and npm as well as the sass and js compilation.
  • The way the js build works currently for local development is that the component js is temporarily copied into a folder local to the designs clientlib directory so they can be injected into the full js build. In AEM this probably wouldn't happen this way (TBD).
